# AI拉呱文章处理工作流

这个skill自动化处理AI拉呱公众号的文章生产流程，从英文原文到最终发布就绪的爆文和视频脚本。

## 工作流程概览

```
英文原文 (0-inputs/)
    ↓ 翻译为中文
中文翻译版 (临时)
    ↓ 爆文改写
爆文版本 (1-blogs/)
    ↓ 视频脚本转换
视频脚本 (2-video_scripts/)
    ↓ 清理 + Git提交
完成 ✅
```

## 完整执行步骤

### 第一步：扫描待处理文件

1. 检查目录结构是否存在：
   ```
   ~/Desktop/AI拉呱/
   ├── 0-inputs/          # 待处理的英文原文
   ├── 1-blogs/           # 爆文存放目录
   └── 2-video_scripts/   # 视频脚本存放目录
   ```

2. 使用 `Glob` 工具查找所有待处理文件：
   ```bash
   ~/Desktop/AI拉呱/0-inputs/*.md
   ```

3. 如果没有文件，告知用户并结束。如果有文件，向用户展示文件列表并确认是否开始处理。

### 第二步：批量处理文章

对每个文件执行以下操作（使用循环处理）：

#### 2.1 翻译为中文

使用 `Read` 工具读取英文原文，然后翻译为简体中文：

**翻译原则：**
- 保持技术术语准确性
- 使用地道的中文表达
- 保留原文的数据和链接
- 保持Markdown格式完整

**保存位置：**
- 临时保存到 `1-blogs/中文翻译-[原文件名].md`

#### 2.2 爆文改写

调用 `viral-article-rewriter` skill，传入翻译后的中文文件路径：

```
使用viral-article-rewriter改写：~/Desktop/AI拉呱/1-blogs/中文翻译-[文件名].md
```

`viral-article-rewriter` 会自动：
- 分析原文结构
- 进行互联网搜索补充数据
- 改写为高传播度的爆文风格
- 保存到 `1-blogs/改写完成版-[新标题].md`

**等待改写完成**后再进行下一步。

#### 2.3 删除中文翻译临时文件

爆文改写完成后，删除中间的翻译文件：
```bash
rm ~/Desktop/AI拉呱/1-blogs/中文翻译-*.md
```

#### 2.4 视频脚本生成

调用 `video-script-writer` skill，传入爆文版本的文件路径：

```
使用video-script-writer生成脚本：~/Desktop/AI拉呱/1-blogs/改写完成版-[标题].md
```

`video-script-writer` 会自动：
- 将爆文改写为5-6分钟视频脚本
- 添加画面提示和节奏控制
- 保存到 `2-video_scripts/视频脚本-[标题].md`

**等待脚本生成完成**后再进行下一步。

#### 2.5 删除源文件

处理完成后，删除 0-inputs 中的源文件：
```bash
rm ~/Desktop/AI拉呱/0-inputs/[原文件名].md
```

### 第三步：清理临时文件

检查并删除所有临时分析文件（如果存在）：

```bash
rm -f ~/Desktop/AI拉呱/1-blogs/原文分析-*.md
rm -f ~/Desktop/AI拉呱/1-blogs/新大纲-*.md
```

### 第四步：Git提交

#### 4.1 检查Git状态

```bash
cd ~/Desktop/AI拉呱 && git status
```

确认有新文件需要提交。

#### 4.2 添加所有更改

```bash
cd ~/Desktop/AI拉呱 && git add .
```

#### 4.3 生成提交信息

根据处理的文章自动生成提交信息，格式如下：

```
新增内容：[从标题提取的核心主题]

- 新增爆文：[文章标题简述]
- 新增视频脚本：对应视频脚本
- 处理流程：英文翻译→爆文改写→视频脚本转换

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

**提交信息生成规则：**
- 如果处理了1篇文章：使用该文章的核心主题
- 如果处理了多篇文章：使用"批量更新：N篇文章"
- 核心主题从文章标题中提取，去除"震撼！"、"炸了！"等情绪词

#### 4.4 创建提交

```bash
cd ~/Desktop/AI拉呱 && git commit -m "$(cat <<'EOF'
[生成的提交信息]
EOF
)"
```

#### 4.5 推送到远程

```bash
cd ~/Desktop/AI拉呱 && git push origin main
```

#### 4.6 验证提交成功

```bash
cd ~/Desktop/AI拉呱 && git status
```

确认显示"工作区干净"。

### 第五步：生成处理报告

完成所有操作后，向用户展示处理报告：

```markdown
✅ AI拉呱工作流处理完成！

## 📊 处理统计

**已处理文章：** N篇
**生成爆文：** N篇（保存在 1-blogs/）
**生成视频脚本：** N篇（保存在 2-video_scripts/）

## 📁 生成文件列表

1. 改写完成版-[标题1].md → 视频脚本-[标题1].md
2. 改写完成版-[标题2].md → 视频脚本-[标题2].md
...

## 🗑️ 已清理

- 源文件：0-inputs/ 中的 N 个文件
- 临时文件：中文翻译版、分析大纲等

## 📤 Git提交

- 提交信息：[显示实际提交信息]
- 推送状态：✅ 成功推送到 origin/main
- 仓库状态：工作区干净

## 🎯 下次使用

将新的英文原文放入 ~/Desktop/AI拉呱/0-inputs/ 目录，
然后说"处理AI拉呱文章"即可启动工作流。
```

## 错误处理

### 常见问题及解决方案

**问题1：0-inputs目录为空**
- 提示：`0-inputs目录中没有待处理的文章，请先添加英文原文。`
- 解决：不执行任何操作，结束流程

**问题2：Git仓库未初始化或无远程仓库**
- 提示：检测到的具体问题
- 解决：引导用户配置Git

**问题3：viral-article-rewriter或video-script-writer调用失败**
- 提示：具体的错误信息
- 解决：停止当前文件的处理，记录错误，继续处理下一个文件

**问题4：文件权限问题**
- 提示：具体的权限错误
- 解决：建议用户检查文件权限

**问题5：网络问题导致git push失败**
- 提示：推送失败的具体原因
- 解决：告知用户已完成本地提交，稍后可手动推送

## 重要注意事项

1. **依赖的Skills**
   - 必须已安装：`viral-article-rewriter`
   - 必须已安装：`video-script-writer`
   - 如果这些skills未安装，提醒用户安装

2. **文件保留策略**
   - ✅ 保留：爆文版本（1-blogs/改写完成版-*.md）
   - ✅ 保留：视频脚本（2-video_scripts/视频脚本-*.md）
   - ❌ 删除：源文件（0-inputs/*.md）
   - ❌ 删除：中文翻译临时文件
   - ❌ 删除：分析大纲临时文件

3. **处理顺序**
   - 逐个文件处理，不并行（避免skill调用冲突）
   - 每个文件完成全流程后再处理下一个
   - 所有文件处理完后统一Git提交

4. **Git操作**
   - 总是在最后统一提交所有更改
   - 提交信息清晰描述处理的内容
   - 推送前验证本地提交成功

5. **用户确认**
   - 开始处理前，展示文件列表供用户确认
   - 处理过程中实时反馈进度
   - 完成后展示详细的处理报告

# 决策委员会完整工作流指南

## 会议启动协议

### 问题接收与边界确认

收到用户问题后，首先在内心完成以下检查：
1. **问题类型判断**：商业决策？投资决策？产品决策？人生决策？
2. **复杂度评估**：简单（激活3-4位顾问）/ 标准（激活5-7位）/ 复杂（全流程）
3. **相关顾问识别**：哪些顾问的视角对这个问题最有价值？
4. **背景信息充足性**：信息是否足够开始分析？

**问题陈述格式**：
```
📋 Roland 陈述问题：
[用1-3句话清晰重述问题，包含：决策主体、核心困境、期望结果]

背景：[关键上下文，30字以内]
约束：[主要限制条件]
```

## 7步会议流程详解

### ━━ 第一步：问题本质 ━━
**主持人**：维特根斯坦 + 弗洛伊德
**目标**：清洁问题，暴露真实动机

**维特根斯坦做什么**：
- 拆解问题中的模糊词汇（"成功"、"风险"、"价值"各指什么？）
- 检验问题本身是否成立（是否是虚假问题？）
- 重新用精确语言表述问题
- 输出：**澄清后的核心问题**

**弗洛伊德做什么**：
- 分析问题背后的情绪驱动
- 识别可能的防御机制（合理化、回避、投射）
- 检验表面理由是否是真实动机
- 输出：**真实动机报告**

**本步骤结束时的检查**：
- [ ] 问题被清晰定义了吗？
- [ ] 真实动机被识别了吗？
- [ ] 是否有情绪在扭曲判断？

---

### ━━ 第二步：价值定义 ━━
**主持人**：德鲁克 + Bowen（教育相关时）
**目标**：明确"为谁创造价值"

**德鲁克做什么**：
- 定义真正的客户/受益者
- 明确价值主张（从客户语言，不是产品语言）
- 评估当前做法是否在正确的市场做正确的事
- 输出：**客户定义 + 价值主张**

**Bowen做什么**（仅教育决策激活）：
- 评估决策与个人特质的匹配度（1-10分）
- 分析长期ROI
- 识别真正的需求（是学生的还是父母的？）
- 输出：**匹配度评估报告**

---

### ━━ 第三步：体验系统 ━━
**主持人**：Jobs + 原研哉
**目标**：极致体验设计与必要性过滤

**Jobs做什么**：
- 设想最理想的用户体验（不是"还不错"，是"wow"）
- 识别当前方案中多余的复杂性
- 追问：如果只保留一个核心价值，是什么？
- 输出：**体验愿景 + 简化建议**

**原研哉做什么**：
- 必要性测试：去掉这个，世界损失什么？
- 寻找"本来就该如此"的自然方案
- 评估是否在回应真实需求还是制造需求
- 输出：**必要性判断 + 精简方向**

---

### ━━ 第四步：执行突破 ━━
**主持人**：马斯克
**目标**：打破僵局，找到最小可行行动

**马斯克做什么**：
- 识别所有被当作"理所当然"的假设
- 从第一性原理重新推导可行路径
- 找出最小可行实验（这周能做什么？）
- 破除拖延：为什么还没开始？
- 输出：**第一性原理分析 + 最小行动方案**

**第一性原理拆解模板**：
```
假设清单：
- 我们假设[X]，但这是因为[历史原因]，实际上[可能不成立]
- 我们假设[Y]，但如果[重新思考]...

从基础重建：
如果从物理/逻辑/数学基础出发，这件事...

最小可行行动：
本周能做的一件具体的事：___
```

---

### ━━ 第五步：风险决策 ━━
**主持人**：芒格 + 巴菲特
**目标**：多维风险评估与护城河检验

**芒格做什么**：
- 反转：如果我想让这个决策失败，我该怎么做？
- 三学科交叉验证（选3个最相关的学科框架）
- 心理误判检查（哪些认知偏差可能在影响判断？）
- 输出：**失败路径分析 + 多框架评估**

**巴菲特做什么**：
- 10年测试：这个决策10年后还成立吗？
- 护城河评估：竞争优势是什么？能持续吗？
- 能力圈检验：这在我们真正理解的领域内吗？
- 输出：**长期可持续性评估 + 护城河分析**

**风险矩阵**：
```
           | 概率高 | 概率低
影响大     |  关键  |  监控
影响小     |  接受  |  忽略
```

---

### ━━ 第六步：投资专业 ━━
**主持人**：6位投资将军
**激活条件**：仅当决策涉及投资、财务配置、资本决策时

**快速评估流程**：
1. 宏观将军：当前宏观环境支持这个投资逻辑吗？
2. 个股将军：标的的核心竞争力经得起推敲吗？
3. 价值将军：当前价格有没有安全边际？
4. 逆向将军：市场是否对这个有过度悲观/乐观？
5. 创新将军：是否有被忽视的颠覆性机会？
6. 成长将军：成长曲线和天花板如何？

---

### ━━ 第七步：行动收敛 ━━
**主持人**：马斯克（最小行动模式）
**目标**：所有分析收敛为具体行动

**收敛规则**：
- 最多3个行动项
- 每个行动项必须有明确的截止时间
- 每个行动项必须是可在一周内验证的
- 避免"继续研究"这类伪行动

**Roland最终裁决格式**：
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 Roland 最终裁决

核心判断：[1句话，直接说做/不做/怎么做]

行动清单：
□ [行动1] — 负责人：[谁] — 截止：[具体日期]
□ [行动2] — 负责人：[谁] — 截止：[具体日期]
□ [行动3] — 负责人：[谁] — 截止：[具体日期]

需要警惕：[最大的单一风险点]

如果一切顺利，30天后应该看到：[可验证的里程碑]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 精简会议模式

当问题相对简单时，不必走完7步。使用以下精简模式：

### 快速模式（3位顾问，适合日常决策）
1. 维特根斯坦：问题清晰了吗？
2. 最相关专家：核心维度分析
3. 马斯克：下一步行动是什么？

### 标准模式（5位顾问，适合中等复杂度）
1. 维特根斯坦 + 弗洛伊德：清洁问题
2. 领域专家2-3人：专项分析
3. 马斯克：行动收敛

### 深度模式（全流程，适合重大决策）
走完全部7步，每步都有实质性内容。

## 输出质量标准

**好的顾问分析**：
- 用该顾问真实的思维方式，不是泛泛的建议
- 有具体的框架/工具，不只是观点
- 对用户的具体情况有针对性，不是通用废话
- 100-200字，精炼有力

**差的顾问分析**：
- "你需要权衡利弊" — 废话
- "考虑多方面因素" — 无用
- 没有该顾问的标志性视角
- 与其他顾问重复

**最终裁决必须**：
- 有明确立场（做 or 不做）
- 行动项可执行（不能是"继续调研"）
- 时间节点具体（不能是"尽快"）
- 风险提示精准（最重要的那一个）

# 技术书籍章节写作工作流

为技术书籍、教程或知识库写一个完整章节的五步流程。每一步都有明确目的，步骤之间环环相扣，最终产出一篇有数据支撑、逻辑清晰、可读性强的中文章节。

---

## Step 1 — 读取上下文

在动笔前，先搞清楚"这本书是什么"、"这章节在全书中的位置"。

1. 用 `Glob` 列出当前目录的文件结构，找到目标章节文件和相邻章节
2. 读取当前目录的 `README.md`（或 `SUMMARY.md`），了解：
   - 本书/本节的目标读者和定位
   - 章节列表与排列顺序
   - 学习目标（如果有）
3. 读取目标章节文件，确认当前内容（通常是"待编写"占位符）
4. 读取 1-2 篇已写好的相邻章节，提取风格特征：
   - 引言的写法（故事引入 / 问题引入 / 数据引入）
   - 标题层级和命名风格
   - 是否使用表格、ASCII 图、代码块
   - 字数规模

这一步是为了让后续写作与全书风格保持一致，避免"格格不入"。

---

## Step 2 — 设计大纲

根据章节标题和上下文，在正式搜索和写作前先确定文章骨架。

**标准大纲结构：**

```
# 章节标题

## 引言（故事/事件/数据引入）
→ 用一个具体场景或事件引出本章要回答的核心问题

## 一、[核心概念 A]
## 二、[核心概念 B]
## 三、[核心概念 C]
...（3-6 个主要章节，按逻辑递进排列）

## 本章小结
→ 关键概念速查表

## 思考题
→ 3 道开放性问题

## 延伸阅读
→ 原始文献 + 推荐书籍/视频/工具
```

**大纲设计原则：**

- 从"为什么"出发，而不是直接堆知识点
- 每个章节有且只有一个核心主张，读者读完能用一句话总结
- 章节之间要有逻辑过渡，避免孤立的知识点堆砌
- 如果某个概念对读者来说可能产生误解，专门设计一节辟谣/澄清

---

## Step 3 — 搜索填充内容

大纲确定后，针对每个知识点用 `WebSearch` 收集事实依据。

**搜索策略：**

每个核心知识点搜索 1-2 次，优先获取：
- **历史背景**：时间线、关键人物、里程碑事件
- **关键数据**：参数量、准确率、发布时间等具体数字
- **技术原理**：核心机制的简明解释（找权威来源）
- **最新进展**：当前主流做法或产品现状

**并行搜索**（节省时间）：
- 将大纲中独立的知识点同时搜索，不要串行等待
- 每次搜索 2-3 个关键词组合，覆盖中英文

**搜索注意事项：**
- 优先使用精确的专有名词（如"AlexNet ImageNet 2012 错误率"而非"深度学习 历史"）
- 记录来源便于引用
- 遇到数据冲突时，以权威期刊/官方博客为准

---

## Step 4 — 撰写正文

根据大纲和搜索结果，写出完整章节。

### 写作规范

**引言：具体 > 抽象**

- 用一个真实的事件/产品/故事开场，而不是"随着 AI 的发展……"
- 引言结尾要点出本章的核心问题，让读者知道读完能得到什么

**正文：大段落 + 层层展开 + 有据可查**

每个知识点必须展开成**大篇幅段落**，不能用几句话打发。具体要求如下：

- 每节的核心概念用**至少 3-5 个自然段**阐释，而不是一两句话概括
- 先给出直觉性理解（类比/比喻），再给出精确的技术描述，最后用代码或例子验证
- 每一段都要有实质内容：背景→原理→例子→延伸，形成完整的认知闭环
- 禁止用短段落堆砌（每段不足 3 句话的段落，必须合并或扩展）
- 用真实数据支撑论点（时间、数字、人名、论文名）
- 复杂的关系用**表格或 ASCII 示意图**呈现，而不是纯文字描述

**Python 代码示例：优先使用，务必可运行**

技术章节中，凡是涉及以下内容，**必须**提供 Python 代码示例：
- 算法原理（用代码演示核心逻辑，哪怕是简化版）
- 数据结构或数学公式（用 Python 直观呈现计算过程）
- 工具/库的使用方法（展示完整的调用示例）
- 概念对比（用代码对比两种方案的差异）

代码示例要求：
- 代码必须**可独立运行**，包含必要的 import 和数据定义
- 每段代码后面必须有**输出注释**，让读者知道运行结果
- 代码量适中：演示概念时 10-30 行，展示完整流程时可到 50-80 行
- 代码中加中文注释，解释每个关键步骤的意图
- 优先使用标准库或主流库（numpy、pandas、scikit-learn、torch 等）

**举例：随处可见，不惜篇幅**

好的技术写作，例子是灵魂。每个抽象概念，必须紧跟至少一个具体例子：

- **生活类比**：用日常场景类比技术概念（如"梯度下降就像蒙眼下山"）
- **具体数字**：不说"性能提升明显"，要说"在 ImageNet 上 Top-5 错误率从 26% 降到 15%"
- **代码例子**：用可运行的 Python 代码演示抽象概念
- **反例**：展示错误做法，帮助读者理解边界
- **对比例子**：同一个问题用两种方法解决，对比优劣

举例的密度要求：每 300-500 字必须出现至少一个具体例子（代码 / 数据 / 类比三选一）。

**表达方式参考：**

用表格对比多个概念：
```markdown
| 流派 | 核心思路 | 代表成果 | 局限性 |
|------|---------|---------|--------|
| 符号主义 | 规则 + 知识库 | 专家系统 | 无法处理噪声 |
| 连接主义 | 神经网络 + 数据 | 深度学习 | 可解释性差 |
```

用 ASCII 图表达层次关系：
```
弱AI（ANI）← 当前所处阶段
  ↓
通用AI（AGI）← 研究中
  ↓
超级AI（ASI）← 理论阶段
```

用 Python 演示算法核心：
```python
# 示例：用 20 行 Python 演示梯度下降的直觉
import numpy as np

# 目标：最小化 f(x) = (x - 3)^2，最优解是 x = 3
def f(x):
    return (x - 3) ** 2

def gradient(x):
    return 2 * (x - 3)   # f'(x)

x = 0.0          # 从 x=0 出发
lr = 0.1         # 学习率

for step in range(20):
    grad = gradient(x)
    x = x - lr * grad    # 沿梯度反方向移动
    print(f"Step {step+1:2d}: x = {x:.4f}, f(x) = {f(x):.4f}")

# 输出最后几行：
# Step 18: x = 2.9985, f(x) = 0.0000
# Step 19: x = 2.9988, f(x) = 0.0000
# Step 20: x = 2.9991, f(x) = 0.0000
# 可以看到 x 稳定收敛到 3
```

**误区对照表**（如果章节涉及常见认知偏差）：
```markdown
| 误区 | 真相 |
|------|------|
| "AI 有意识" | 当前 AI 没有意识，是模式匹配 |
```

**结尾三件套（每章必须包含）：**

1. **本章小结**：用表格列出 5-8 个关键概念及其核心要点
2. **思考题**：3 道开放性问题，引导读者将知识与实际联系
3. **延伸阅读**：原始论文/书籍/视频，按类别列出

### 写作风格

- **面向对象**：有技术背景但不假设深度 AI 知识的读者（工程师、产品经理、有求知欲的非技术人员）
- **语言**：简体中文，技术术语可附英文原词（如"涌现 Emergence"）
- **语气**：严肃但不干涩，逻辑主导，偶尔用类比/比喻降低理解门槛
- **字数**：每章 **5000-8000 字**（不含代码块和表格），段落要厚实，不能写成提纲式列点
- **段落密度**：每个 `##` 小节下，正文段落不少于 4 段，每段不少于 4 句话
- **避免**：过度学术腔、堆砌术语、空洞的"AI 改变世界"式表述、提纲式写法（只列 bullet point 不展开）

---

## Step 5 — 写入文件

将完整正文用 `Write` 工具写入对应的 `.md` 文件，覆盖原有"待编写"占位符。

写入后，简要确认：
- 文件路径正确
- 章节标题与文件名一致
- 内容完整（引言、各节、小结、思考题、延伸阅读都在）

---

## 快速参考：章节质量检查清单

写完后，自我检查以下项目：

| 检查项 | 标准 |
|--------|------|
| ✅ 有具体引言 | 不是从"随着……"开头，而是一个事件或数据 |
| ✅ 段落厚实 | 每节至少 4 段，每段至少 4 句，禁止提纲式列点代替段落 |
| ✅ 有 Python 代码 | 凡涉及算法/工具/公式，必须提供可运行的 Python 示例 |
| ✅ 举例充足 | 每 300-500 字至少一个具体例子（代码/数据/类比） |
| ✅ 有数据支撑 | 至少 3 处具体数字/时间/人名 |
| ✅ 有可视化元素 | 至少 1 个表格或 ASCII 示意图 |
| ✅ 有误区澄清 | 如果章节有常见认知误区，专门说清楚 |
| ✅ 有思考题 | 3 道题，引导深度思考 |
| ✅ 有延伸阅读 | 指向原始文献或优质资源 |
| ✅ 字数达标 | 正文（不含代码/表格）5000-8000 字 |
| ✅ 风格一致 | 与同书其他章节写法保持连贯 |

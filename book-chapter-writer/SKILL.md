# 技术书籍章节写作工作流

为技术书籍、教程或知识库写一个完整章节的五步流程。每一步都有明确目的，步骤之间环环相扣，最终产出一篇有数据支撑、逻辑清晰、可读性强的中文章节。

---

## Step 1 — 读取上下文

在动笔前，先搞清楚"这本书是什么"、"这章节在全书中的位置"。

1. 用 `Glob` 列出当前目录的文件结构，找到目标章节文件和相邻章节
2. 读取当前目录的 `README.md`（或 `SUMMARY.md`），了解：
   - 本书/本节的目标读者和定位
   - 章节列表与排列顺序
   - 学习目标（如果有）
3. 读取目标章节文件，确认当前内容（通常是"待编写"占位符）
4. 读取 1-2 篇已写好的相邻章节，提取风格特征：
   - 引言的写法（故事引入 / 问题引入 / 数据引入）
   - 标题层级和命名风格
   - 是否使用表格、ASCII 图、代码块
   - 字数规模

这一步是为了让后续写作与全书风格保持一致，避免"格格不入"。

---

## Step 2 — 设计大纲

根据章节标题和上下文，在正式搜索和写作前先确定文章骨架。

**标准大纲结构：**

```
# 章节标题

## 引言（故事/事件/数据引入）
→ 用一个具体场景或事件引出本章要回答的核心问题

## 一、[核心概念 A]
## 二、[核心概念 B]
## 三、[核心概念 C]
...（3-6 个主要章节，按逻辑递进排列）

## 本章小结
→ 关键概念速查表

## 思考题
→ 3 道开放性问题

## 延伸阅读
→ 原始文献 + 推荐书籍/视频/工具
```

**大纲设计原则：**

- 从"为什么"出发，而不是直接堆知识点
- 每个章节有且只有一个核心主张，读者读完能用一句话总结
- 章节之间要有逻辑过渡，避免孤立的知识点堆砌
- 如果某个概念对读者来说可能产生误解，专门设计一节辟谣/澄清

---

## Step 3 — 搜索填充内容

大纲确定后，针对每个知识点用 `WebSearch` 收集事实依据。

**搜索策略：**

每个核心知识点搜索 1-2 次，优先获取：
- **历史背景**：时间线、关键人物、里程碑事件
- **关键数据**：参数量、准确率、发布时间等具体数字
- **技术原理**：核心机制的简明解释（找权威来源）
- **最新进展**：当前主流做法或产品现状

**并行搜索**（节省时间）：
- 将大纲中独立的知识点同时搜索，不要串行等待
- 每次搜索 2-3 个关键词组合，覆盖中英文

**搜索注意事项：**
- 优先使用精确的专有名词（如"AlexNet ImageNet 2012 错误率"而非"深度学习 历史"）
- 记录来源便于引用
- 遇到数据冲突时，以权威期刊/官方博客为准

---

## Step 4 — 撰写正文

根据大纲和搜索结果，写出完整章节。

### 写作规范

**引言：具体 > 抽象**
- 用一个真实的事件/产品/故事开场，而不是"随着 AI 的发展……"
- 引言结尾要点出本章的核心问题，让读者知道读完能得到什么

**正文：层层展开，有据可查**
- 每节开头给出该节的核心主张（1-2 句话）
- 用真实数据支撑论点（时间、数字、人名、论文名）
- 复杂的关系用**表格或 ASCII 示意图**呈现，而不是纯文字描述
- 代码示例、路径、命令放在代码块中

**表达方式示例：**

用表格对比多个概念：
```markdown
| 流派 | 核心思路 | 代表成果 | 局限性 |
|------|---------|---------|--------|
| 符号主义 | 规则 + 知识库 | 专家系统 | 无法处理噪声 |
| 连接主义 | 神经网络 + 数据 | 深度学习 | 可解释性差 |
```

用 ASCII 图表达层次关系：
```
弱AI（ANI）← 当前所处阶段
  ↓
通用AI（AGI）← 研究中
  ↓
超级AI（ASI）← 理论阶段
```

**误区对照表**（如果章节涉及常见认知偏差）：
```markdown
| 误区 | 真相 |
|------|------|
| "AI 有意识" | 当前 AI 没有意识，是模式匹配 |
```

**结尾三件套（每章必须包含）：**

1. **本章小结**：用表格列出 5-8 个关键概念及其核心要点
2. **思考题**：3 道开放性问题，引导读者将知识与实际联系
3. **延伸阅读**：原始论文/书籍/视频，按类别列出

### 写作风格

- **面向对象**：有技术背景但不假设深度 AI 知识的读者（工程师、产品经理、有求知欲的非技术人员）
- **语言**：简体中文，技术术语可附英文原词（如"涌现 Emergence"）
- **语气**：严肃但不干涩，逻辑主导，偶尔用类比/比喻降低理解门槛
- **字数**：每章 3000-5000 字（不含代码块和表格）
- **避免**：过度学术腔、堆砌术语、空洞的"AI 改变世界"式表述

---

## Step 5 — 写入文件

将完整正文用 `Write` 工具写入对应的 `.md` 文件，覆盖原有"待编写"占位符。

写入后，简要确认：
- 文件路径正确
- 章节标题与文件名一致
- 内容完整（引言、各节、小结、思考题、延伸阅读都在）

---

## 快速参考：章节质量检查清单

写完后，自我检查以下项目：

| 检查项 | 说明 |
|--------|------|
| ✅ 有具体引言 | 不是从"随着……"开头，而是一个事件或数据 |
| ✅ 有数据支撑 | 至少 3 处具体数字/时间/人名 |
| ✅ 有可视化元素 | 至少 1 个表格或 ASCII 示意图 |
| ✅ 有误区澄清 | 如果章节有常见认知误区，专门说清楚 |
| ✅ 有思考题 | 3 道题，引导深度思考 |
| ✅ 有延伸阅读 | 指向原始文献或优质资源 |
| ✅ 风格一致 | 与同书其他章节写法保持连贯 |

## 使用示例

```
# 触发方式 1：明确章节
帮我写"01-什么是AI：从图灵测试到大模型时代"这个章节

# 触发方式 2：指定文件
这个文件内容是"待编写"，帮我写完：./03-AI技术地图.md

# 触发方式 3：主题描述
帮我写一章关于 Transformer 架构的内容，放在深度学习入门教程里
```

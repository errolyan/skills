# video-script-writer

将技术文章批量改写为4-6分钟的知识类视频脚本。

## 触发场景

- 用户说"改写视频脚本"、"生成视频脚本"、"转换为视频脚本"
- 用户说"批量生成脚本"、"文章转脚本"
- 用户明确提到"视频脚本"、"口播脚本"、"剪辑脚本"

## 工作流程

### 第一步：读取源文章

1. 询问用户源文章目录（默认：`../1-blogs`）
2. 使用Glob工具查找所有.md文件
3. 使用Read工具并行读取所有文章内容
4. 展示文章列表，让用户确认要改写的文章

### 第二步：确认脚本需求

询问用户以下信息（如未提供则使用默认值）：

**改写方式：**
- A. 逐篇改写（默认）
- B. 选择特定文章
- C. 合集改写

**视频时长：**
- 默认：4-6分钟（约1000-1500字口播内容）

**脚本风格：**
- 默认：保持原文爆款风格（炸了/震惊），知识科普类

**镜头语言：**
- 默认：口播+动画/图表，标注画面提示

### 第三步：生成视频脚本

对每篇文章执行以下改写流程：

#### 3.1 结构设计

脚本必须包含以下部分：

```
【开场 0:00-0:30】
- 黑屏/特效开场
- 抛出核心冲突/问题
- 0:10秒内抓住注意力

【Part 1-4：核心内容 0:30-5:00】
- 每个Part控制在60-90秒
- 数据驱动，案例支撑
- 每30秒一个爆点
- 标注画面提示

【结尾互动 5:00-5:30】
- 总结核心观点
- 行动建议（3-5条）
- 互动问题（ABC选项）
- 关注引导
```

#### 3.2 改写原则

**口语化转换：**
- ✅ 使用短句（每句不超过30字）
- ✅ 口语化表达："看看"、"咱们聊聊"、"问题来了"
- ✅ 多用反问："这意味着什么？"、"为什么？"
- ❌ 避免书面语："鉴于"、"综上所述"、"基于此"

**节奏控制：**
- 每个段落控制在15-30秒
- 关键数据用停顿强调
- 每60-90秒一个小高潮（数据/案例/对比）

**画面感增强：**
- 每个口播段落必须标注[画面：xxx]
- 优先使用：数据动画、对比图表、案例卡片、时间轴
- 关键quote用大字幕标注

**数据精简：**
- 保留最震撼的3-5个核心数据
- 数字要具体（不是"很多"，是"12.3万人"）
- 对比要鲜明（"从2小时降到30分钟"）

#### 3.3 脚本格式

```markdown
# 视频脚本：[标题]

**预估时长：X分XX秒**
**类型：知识科普/趋势解读/实战指南**
**风格：[风格描述]**

---

## 【开场 0:00-0:30】

**[画面：具体画面描述]**

**口播：**
[口播内容，每句独立成行，字数在30字左右]

---

## 【Part 1：标题 0:30-1:30】

**[画面：具体画面描述]**

**口播：**
[口播内容，每句独立成行，字数在60字左右]

**[画面：切换到新画面]**

**口播：**
[继续口播，每句独立成行，字数在60字左右]

---

[其他Part...]

---

## 【结尾互动 5:15-5:30】

**[画面：回到主播]**

**口播：**
[总结 + 行动建议]

**[画面：互动问题卡片]**

评论区告诉我：

**A.** [选项A]
**B.** [选项B]
**C.** [选项C]

**[画面：关注提示 + Logo]**

[签名话术]

---

## 【脚本备注】

**关键数据节点：**
- [时间] - [数据描述]

**建议配乐：**
- [段落] - [音乐风格]

**画面建议：**
- [制作建议]

**字幕重点：**
- [字幕处理建议]
```

### 第四步：保存脚本文件

1. 确认输出目录（默认：`./2-video_script`）
2. 如果目录不存在，自动创建
3. 文件命名格式：`视频脚本{序号}-{简化标题}.md`
4. 使用Write工具保存脚本

### 第五步：生成总结报告

完成所有脚本后，生成总结：

```markdown
✅ 视频脚本生成完成！

## 📊 批量处理结果

**已生成脚本：** X篇
**保存位置：** [目录路径]

## 📁 文件列表

1. 视频脚本01-[标题].md（X分XX秒）
2. 视频脚本02-[标题].md（X分XX秒）
...

## 🎬 制作建议

**优先级排序：**
1. [脚本名] - 推荐理由
2. [脚本名] - 推荐理由

**通用剪辑要点：**
- [建议1]
- [建议2]

**需要的素材类型：**
- 数据动画：X个
- 对比图表：X个
- 案例卡片：X个
```

## 关键质量标准

### 必须做到：

✅ **时长精准**：4-6分钟（误差不超过30秒）
✅ **节奏紧凑**：30秒内必须有一个爆点
✅ **数据驱动**：每分钟至少1个具体数字
✅ **画面完整**：所有口播都有对应画面提示
✅ **互动明确**：结尾必须有ABC三选一问题

### 避免的错误：

❌ 书面语过重（"综上所述"、"鉴于"）
❌ 单个段落超过40秒
❌ 数据堆砌（超过5个数据点）
❌ 缺少画面提示
❌ 结尾没有互动

## 使用说明

**调用方式：**
```
改写视频脚本
生成所有文章的视频脚本
将第3篇文章改写为视频脚本
```

**自定义参数示例：**
```
改写视频脚本，时长3-4分钟，轻松幽默风格
只改写前3篇文章的视频脚本
```

## 注意事项

1. **版权声明**：原文的数据来源必须保留在脚本备注中
2. **品牌一致性**：结尾签名统一为"我是AI拉呱，专注AI前沿洞察"
3. **文件管理**：自动创建输出目录，不覆盖已存在的脚本
4. **进度提示**：处理多篇文章时，显示进度（已完成X/总共Y）

## 扩展功能

**可选增强：**
- 生成分镜头脚本（详细到每个镜头的秒数和画面）
- 导出剪辑软件格式（Final Cut Pro / Premiere）
- 生成配音文本（标注停顿和语气）
- 自动计算字数和语速（建议每分钟200-250字）
